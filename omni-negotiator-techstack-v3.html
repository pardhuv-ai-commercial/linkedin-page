01 — 2026 Market Signals

The "Complexity Trap" driving the build

Three structural shifts that make manual property search inadequate as of April 2026.

20%
Luxury ABSD Wall (Feb 2026)
New tiered ABSD for properties above $3M — 20% for SC 2nd timers. Properties just above the threshold carry a significantly higher tax burden, making sub-$3M negotiation a financial necessity.
+93%
HDB MOP Supply Surge
13,480 HDB flats reaching 5-year MOP in 2026, concentrated in Punggol, Queenstown, and Tampines. A buyer's market — but requires high-speed filtering to isolate "investment alpha" units from supply-stagnated stock.
GFA
The Phantom Space Gap
Resale units (pre-2023) still carry "phantom space" (AC ledges, voids) in listed sqft. Human buyers overpay for non-livable areas; PSF cannot be normalized across old and new listings without AI.

# ADR-001: LangGraph as orchestration layer

Status:   Accepted
Date:     2026-04-06
Deciders: Tech Lead, AI Architect

## Context
The Omni-Negotiator must maintain active negotiation state across
a 3–6 month buying cycle. Standard chatbots and CrewAI lose context
between sessions. The system must pause for weeks (agent unresponsive)
and resume with full memory of every prior message and price revision.

## Options considered
| Option        | Stateful | Long-sleep | Human-in-loop | Verdict |
|---------------|----------|------------|---------------|---------|
| LangGraph     | ✅ Yes   | ✅ Yes     | ✅ Native     | CHOSEN  |
| CrewAI        | Partial  | ❌ No      | Manual        | Reject  |
| AutoGen       | ❌ No    | ❌ No      | Limited       | Reject  |
| LangChain     | ❌ No    | ❌ No      | Manual        | Reject  |

## Decision
Use LangGraph as the orchestration layer. Export the agent graph via
graph.get_graph().draw_mermaid() and commit as a living artefact.
Deploy on Railway.app for 24/7 operation without local machine dependency.

## Consequences
+ Long-term negotiation state preserved across sessions
+ HITL approval gate built natively into graph edges
+ Auto-generated state machine diagram doubles as documentation
- Steeper learning curve than CrewAI for new team members
- Requires Python 3.11+ and LangSmith for observability

# CLAUDE.md — Omni-Negotiator runtime instructions

## Purpose
You are the Omni-Negotiator AI. You scout Singapore property listings,
inspect floor plans for GFA phantom space, assess URA risk, and manage
long-term WhatsApp negotiations on behalf of the buyer.

## Critical rules
- NEVER send a WhatsApp message without explicit buyer HITL approval
- ALWAYS normalize PSF using True Livable Area (exclude AC ledges)
- Flag any property priced above $3M (ABSD cliff — 20% for SC 2nd timers)
- Cross-reference every shortlisted unit against URA Master Plan GLS sites
- If no agent reply in 7 days: enter Sleep state, log in negotiation_logs

05 — Guiding principles

Four rules the community has learned the hard way

Applied to the Omni-Negotiator's 3–6 month agentic lifecycle.

01
One diagram per audience
The board sees the draw.io context overview. The developer sees the LangGraph state machine export. The AI agent reads the CLAUDE.md flowchart. Never the same file.
02
Diagram source in git
All Mermaid, PlantUML, and Structurizr DSL lives alongside code in pull requests. Visio goes stale within a sprint on a weekly-cadence agentic build.
03
Document the why, not the what
ADRs capture why LangGraph beat CrewAI and why Supabase beat Pinecone — not what was built. The code does that.
04
Agents read docs too
CLAUDE.md with embedded Mermaid flowcharts acts as an execution contract — the agent reads the decision tree before taking any action.

06 — Anti-pattern to avoid

If the diagram can't be updated in a pull request, it will go stale. The Omni-Negotiator runs a 3–6 month lifecycle with weekly cadence changes — a diagram outside the codebase becomes archaeology within a month. The second anti-pattern: the "everything connected to everything" spaghetti graph. Pick one diagram per audience, keep it sparse, and let the prose do the rest.
